# 隐私说明

AI IME 的目标是学习个人拼音误输入习惯，因此它会处理敏感输入数据。任何公开发布都必须把隐私边界讲清楚，并让用户能随时控制。

## 本地数据

默认数据目录：

```text
%LOCALAPPDATA%\AIIME
```

主要文件：

- `ai-ime.db`：纠错事件和规则。
- `keylog.jsonl`：完整键盘日志，仅在开启“记录完整本地键盘日志”后写入。
- `learning.log`：学习和部署日志。
- `settings.json`：应用设置。

项目根目录 `.env` 保存模型地址和 API Key，已被 `.gitignore` 忽略。

## 模型上传策略

- OpenAI 兼容接口和中转站：默认只发送纠错事件，不发送完整键盘日志。
- 用户开启“允许云端/中转模型接收完整键入日志”后，才会发送完整键盘日志。
- Ollama 等本地模型：可以默认使用完整键盘日志作为上下文，因为数据仍在本机链路内。
- 上传内容会被包装成 JSON，包括 `events` 和可选 `keylog_entries` 两个部分。模型必须返回标准规则 JSON，本地解析器会拒绝非标准结构。

## 用户应有控制

当前已支持：

- 启停监听。
- 开关完整本地键盘日志。
- 开关完整日志上传。
- 打开记录文件。
- 清理事件和规则的命令行能力。

后续发布前建议补齐：

- 设置界面内一键清空键盘日志。
- 设置界面内导出/删除纠错数据库。
- 应用黑名单。
- 密码框、浏览器隐私页面、远程桌面等敏感上下文的自动暂停。
- 首次启动隐私确认页。

## 开发约束

- 日志和测试输出不得打印 API Key。
- UI 状态只能展示脱敏 key。
- 新增 provider 时必须显式测试“是否发送完整 keylog”的开关行为。
- 发布文档必须说明这是本地输入监听工具，用户需要主动启用并理解风险。
